# Scenario-Based Survey Design

## Purpose

แบบประเมินนี้ออกแบบเพื่ออ่าน **Culture Profile** ผ่านสถานการณ์ทำงานจริง โดยไม่ถามตรง ๆ ว่าองค์กรเป็น Clan, Adhocracy, Market หรือ Hierarchy

## Response Format

ทุกสถานการณ์ตอบ 2 มุมมอง:

1. **Current Culture** — สิ่งที่มักเกิดขึ้นจริงในปัจจุบัน
2. **Desired Culture** — สิ่งที่ควรเกิดขึ้นในอนาคต

ผู้ตอบเลือก 1 จาก 4 ตัวเลือกในแต่ละมุมมอง

## Archetype Mapping

- Clan — Collaboration / Trust / Belonging
- Adhocracy — Innovation / Agility / Experimentation
- Market — Result / Accountability / Customer Impact
- Hierarchy — Clarity / Process / Consistency

## Scenario Coverage

| Scenario | Workplace Dimension |
|---|---|
| S01 | New idea / innovation |
| S02 | Cross-functional project |
| S03 | Urgent issue ownership |
| S04 | Goal setting |
| S05 | Failure and learning |
| S06 | Conflict and decision |
| S07 | Customer / audience shift |
| S08 | Frontline decision authority |
| S09 | Leadership role |
| S10 | Resource allocation |
| S11 | Recognition / promotion |
| S12 | Workflow transformation |

## Scoring

ตัวเลือกแต่ละข้อแทน Archetype 1 ประเภท

- Archetype score range ต่อมุมมอง = 0–12
- Percentage = score / 12 × 100
- Top profile รองรับ Tie มากกว่า 1 Archetype
- Gap = Desired score - Current score

## Interpretation Guardrails

- ผลรายบุคคลไม่ใช่ Personality Test
- ไม่มี Archetype ใดดีที่สุดในทุกสถานการณ์
- คะแนนสูงไม่ได้แปลว่าดีเสมอ และคะแนนต่ำไม่ได้แปลว่าแย่เสมอ
- ต้องอ่านร่วมกับ Strategic Culture Requirement และบริบทธุรกิจ
- ห้ามสรุป Core Values จาก Rank เพียงอย่างเดียว

## Validation Before Production

แนะนำ Pilot 30–50 คน เพื่อตรวจ:
- ความชัดเจนของภาษา
- เวลาที่ใช้ตอบ
- ตัวเลือกที่คลุมเครือหรือดีเกินจริง
- Response distribution ของแต่ละข้อ
- Internal consistency ราย Archetype
- ความแตกต่างระหว่าง BU / Level ที่สมเหตุสมผล
- Cognitive Interview 5–8 คน เพื่อถามว่าผู้ตอบตีความแต่ละตัวเลือกอย่างไร

แบบประเมินใน Project นี้เป็น Organizational Diagnostic MVP และยังไม่ใช่ Psychometric Instrument ที่ผ่านการรับรอง
